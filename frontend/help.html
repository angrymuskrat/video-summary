Quick operator guide

How to run the stack and use the UI

The frontend is static, the API owns background execution, and the database keeps job and artifact metadata indexed by job ID.

Recommended flow

1. Open Upload
Choose a file, set pipeline flags, and submit the job.
2. Copy the job ID
The backend responds immediately with the tracking identifier.
3. Monitor Status
Use the status page to inspect lifecycle timestamps, errors, and current step.
4. Browse Artifacts
Use the artifacts page to preview transcripts or download PPTX, subtitles, JSON, and video outputs.
5. Let retention cleanup run
Expired jobs and their managed files are removed automatically according to server settings.

Server-side settings

Database
`VIDEO_SUMMARY_DATABASE_URL` controls the SQLAlchemy connection string.
Storage
`VIDEO_SUMMARY_STORAGE_ROOT` points to managed upload/output directories.
Retention
`VIDEO_SUMMARY_ARTIFACT_RETENTION_HOURS` defines cleanup lifetime.
OpenAI
`OPENAI_API_KEY`, `OPENAI_MODEL`, and optional `OPENAI_BASE_URL` stay on the backend only.

Summarization mode

The OpenAI-backed summarizer only reads transcript utterances. Slide frames and other modalities are ignored, which keeps the integration simple and testable while matching the current requirement.
